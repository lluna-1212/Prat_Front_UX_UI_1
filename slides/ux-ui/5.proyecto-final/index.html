Plantilla: Proyecto Final

Utiliza esta estructura para planificar tu proyecto integrando React/React Native, diseño de interfaces UX/UI y componentes open-source.

1. Título del Proyecto

[Nombre provisional]

2. Descripción General

  • Objetivo principal del producto.
  • Público objetivo y sus necesidades.
  • Beneficio clave o propuesta de valor.

3. Tecnologías y Arquitectura

  • Frontend Web: React
  • Mobile: React Native
  • Estilos: CSS-in-JS / Styled Components / Tailwind
  • Gestión de Estado: Context API / Redux / Zustand
  • Open-Source: Listar librerías o componentes OSS que integrarás

4. Proceso UX/UI

  1. Investigación de usuarios: entrevistas, encuestas.
  2. Definición de personas y user journeys.
  3. Sketches y wireframes (baja fidelidad).
  4. Prototipos interactivos (alta fidelidad).
  5. Pruebas de usabilidad y ajustes.

5. Diseño de Interfaces

  • Guía de estilo: tipografía, colores, espaciado.
  • Componentes reutilizables: botones, formularios, cards.
  • Sistema de diseño (Storybook / Figma Library).
  • Accesibilidad: roles ARIA, contraste y navegación teclado.

6. Integración Open-Source

  • Seleccionar 1–2 componentes OSS listos para usar (p.ej. calendario, mapa, gráfico).
  • Personalizar estilos y funcionalidad según tu diseño.
  • Contribuir de vuelta: pull request con mejoras o documentación.

7. Estructura de Carpetas

src/
├─ components/
│  ├─ ui/
│  └─ common/
├─ screens/
├─ hooks/
├─ contexts/
├─ services/
├─ assets/
└─ navigation/

8. Cronograma y Entregables

  • Semana 1–2: Investigación y wireframes.
  • Semana 3–4: Prototipo alta fidelidad.
  • Semana 5–6: Configuración proyecto React/React Native.
  • Semana 7–8: Desarrollo de componentes y pantalla principal.
  • Semana 9: Integración OSS y pruebas.
  • Semana 10: Test de usabilidad y ajustes finales.

9. Criterios de Evaluación

  • Calidad del diseño visual y usabilidad.
  • Funcionalidad core implementada en web y móvil.
  • Uso correcto de Hooks y patrones de React/React Native.
  • Integración y contribución a componente open-source.
  • Documentación clara y presentación final.

10. Documentación y Handoff

  • README con descripción, setup y scripts.
  • Guía de estilo y componentes (Storybook/Figma).
  • Instrucciones de despliegue para web y móvil.
